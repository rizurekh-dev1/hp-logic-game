# Design: Harry Potter Logic Game (PWA)

Generated by /office-hours

## Problem Statement
Creating an engaging, mobile-friendly game for a 10-year-old son to help him develop critical thinking skills, without relying on erratic hardware sensors.

## The Hook
An animated sequence where Harry Potter breaks the fourth wall, asking the user directly for help in rescuing his friends from Voldemort. The first challenge is based on the first book (e.g., Devil's Snare or Flying Keys).

## Core Interaction Loop
1. Narrative Hook (High-impact animation)
2. Challenge (Visual/Text logic puzzle, specifically a jigsaw puzzle for the first level)
3. Success & Story Progression

## Recommended Approach
**Approach A: The Web-Based PWA**
We will build a Progressive Web App using React (Vite) and Framer Motion. This provides the fastest path to a playable version, allowing immediate deployment and easy "Add to Home Screen" installation on both Android and iOS. It avoids app store overhead while delivering a native-feeling 60fps experience for puzzles and animations.

## Design System & UI/UX Constraints
- **Visual Identity:** Dark parchment background (`#1A1A1A` with subtle noise). Electric spell-blue (`#00E5FF`) for active states/success. Typography uses a highly readable sans-serif (`Inter`) for puzzles, paired with a magical serif (`Cinzel` or `Playfair Display`) for Harry's dialogue.
- **Micro-interactions:** Dragging a piece lifts it slightly (scale 1.05) with a soft shadow. Correct snaps trigger a brief gold particle burst. Incorrect snaps aggressively return to the tray with a subtle haptic "thud".
- **Accessibility & Controls:** Dual-control scheme. Users can drag-and-drop, OR tap to select a piece (it glows blue) and tap an empty slot to swap it. Minimum touch target of `44x44px` for all interactive elements.
- **Offline Edge Case:** The game requires an intentional offline state ("The Floo Network is down!") featuring a cached, simple mini-game to handle poor connectivity gracefully.

## Constraints & Trade-offs
- No hardware sensor inputs (accelerometer/gyroscope) to avoid accidental spurious inputs.
- As a PWA, it has minor storage limitations compared to true native, but this is acceptable for a lightweight puzzle game.

## Distribution Plan
- Hosted on Vercel or Netlify.
- Distributed via a direct URL.
- Installed on the device via the browser's "Add to Home Screen" feature.

## Engineering Architecture & Constraints
- **State Management:** Strict state machine (via `useReducer`) to handle the scalable game loop: `INITIAL_LOAD` -> `LEVEL_INTRO` -> `PUZZLE_ACTIVE` -> `LEVEL_SUCCESS` -> `NEXT_LEVEL_TRANSITION` (increments `currentLevelIndex`) -> `LEVEL_INTRO`. Global state tracks `currentLevelIndex`, `totalLevels`, and `unlockedLevels`.
- **Performance:** All animations and drag-and-drop movement MUST use hardware-accelerated CSS (`transform: translate3d`) rather than layout-triggering properties (`top/left`) to maintain 60fps on older Android devices.
- **Preloading:** Strict asset preloader. The app stays in `INITIAL_LOAD` until critical assets for the `currentLevel` are cached to prevent jarring mid-animation pop-ins.
- **PWA / Offline Strategy:** Vite PWA Plugin with Workbox using a `CacheFirst` strategy. Caches HTML, CSS, JS, and a lightweight offline fallback state.
- **Testing:** Unit testing (Vitest) for pure logic functions (e.g., puzzle validation, state transitions). Manual QA for drag-and-drop tactile feel.
- **Mobile Rendering & Responsiveness:** 
  - **Viewport Lock:** `<meta name="viewport" content="... user-scalable=no, viewport-fit=cover">` to prevent accidental pinch-zoom during fast taps.
  - **Safe Areas:** CSS `env(safe-area-inset-*)` protects UI elements from camera notches and bottom swipe bars.
  - **Dynamic Scaling:** `height: 100dvh` ensures full screen height without browser bar clipping. The puzzle container uses `width: min(90vw, 400px); aspect-ratio: 1/1;` to gracefully scale across 6.1" to 7" screens while maintaining exact puzzle proportions.
- **Idle / Distraction Protocol:** 
  - **Audio/Auto-play Gate:** The game starts in a `TAP_TO_START` state to explicitly unlock the browser's audio context before the Harry Potter narrative begins.
  - **Looping Hook:** To prevent missed context if distracted, the introductory "fourth-wall break" animation seamlessly loops until the user interacts with the puzzle.
  - **State Persistence & Wake Lock:** Implements the Screen Wake Lock API to prevent the screen from sleeping while thinking. `localStorage` saves the puzzle state on every single piece drop to ensure zero data loss if the app is killed.
- **Analytics (Adapter Pattern):**
  - **Initial Provider:** PostHog (free tier, funnel dashboards out of the box).
  - **Architecture:** A dedicated `src/analytics.js` module is the ONLY file that imports or references any analytics SDK. All other components exclusively call `trackEvent(eventName, properties)` from this adapter. Swapping to Google Analytics 4 (or any other provider) requires changing ONLY `analytics.js`. Zero hunt-and-replace across the rest of the codebase.
  - **Key events to track:** `app_launched`, `tap_to_start`, `level_intro_viewed`, `puzzle_started`, `piece_placed`, `puzzle_solved`, `level_failed`.
